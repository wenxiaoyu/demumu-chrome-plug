# 发布前检查清单

在提交到 Chrome Web Store 之前，请逐项检查以下内容。

## 📦 代码和构建

- [ ] 所有功能已完成并测试通过
- [ ] 没有 console.log 或调试代码
- [ ] 所有 TODO 注释已处理
- [ ] 代码已通过 TypeScript 编译（无错误）
- [ ] 运行 `npm run build` 成功
- [ ] 检查 `dist/` 目录包含所有必要文件
- [ ] `manifest.json` 版本号正确（如 `1.0.0`）
- [ ] `package.json` 版本号与 manifest 一致

## 🔧 配置和权限

- [ ] Firebase 配置已正确设置
- [ ] Firebase 项目已启用必要的服务：
  - [ ] Authentication (Google 登录)
  - [ ] Firestore Database
  - [ ] Security Rules 已配置
- [ ] manifest.json 中的权限说明清晰
- [ ] 只请求必要的权限（不要过度请求）
- [ ] host_permissions 列表正确

## 🎨 图标和资源

- [ ] 16x16 图标存在且清晰 (`src/icons/icon-16.png`)
- [ ] 48x48 图标存在且清晰 (`src/icons/icon-48.png`)
- [ ] 128x128 图标存在且清晰 (`src/icons/icon-128.png`)
- [ ] 所有图标背景透明或适当
- [ ] 准备了 3-5 张截图（1280x800 或 640x400）
- [ ] 截图展示了主要功能
- [ ] 截图清晰美观，无敏感信息

## 📝 文本内容

- [ ] 准备了简短描述（132 字符以内）
- [ ] 准备了详细描述（包含功能介绍、使用方法）
- [ ] 中英文描述都已准备
- [ ] 描述中没有夸大或误导性内容
- [ ] 准备了更新说明（首次发布可写"首次发布"）

## 🔒 隐私和安全

- [ ] 创建了隐私政策页面
- [ ] 隐私政策链接可访问
- [ ] 说明了数据收集和使用方式
- [ ] 说明了每个权限的用途
- [ ] 说明了主机权限的必要性
- [ ] 没有收集不必要的用户数据
- [ ] 数据传输使用 HTTPS

## 🌍 国际化

- [ ] 支持中文（简体）
- [ ] 支持英文（美国）
- [ ] `_locales/zh_CN/messages.json` 完整
- [ ] `_locales/en/messages.json` 完整
- [ ] 所有 UI 文本都使用 `t()` 函数
- [ ] 没有硬编码的文本

## 🧪 测试

- [ ] 在 Chrome 中加载扩展测试
- [ ] 测试所有主要功能：
  - [ ] 敲木鱼功能
  - [ ] HP 计算正确
  - [ ] 联系人添加/编辑/删除
  - [ ] Google 登录
  - [ ] 数据同步
  - [ ] 邮件预览
  - [ ] 语言切换
  - [ ] 设置保存
- [ ] 测试离线模式
- [ ] 测试多设备同步
- [ ] 检查浏览器控制台无错误
- [ ] 检查 Network 面板无失败请求

## 📊 性能

- [ ] 扩展加载速度快（< 1 秒）
- [ ] 没有内存泄漏
- [ ] 没有过度的网络请求
- [ ] 图片已优化（大小合理）
- [ ] 构建产物大小合理（< 5MB）

## 📄 文档

- [ ] README.md 完整且最新
- [ ] 包含安装说明
- [ ] 包含使用说明
- [ ] 包含开发说明
- [ ] 包含贡献指南（如果开源）
- [ ] LICENSE 文件存在

## 🔐 账号准备

- [ ] 有 Google 账号
- [ ] Google 账号已启用两步验证
- [ ] 已注册 Chrome Web Store 开发者账号（$5）
- [ ] 可以访问 Developer Dashboard

## 📦 打包

- [ ] 运行 `npm run build` 无错误
- [ ] 进入 `dist/` 目录
- [ ] 压缩所有文件为 `extension.zip`
- [ ] 确认 zip 根目录包含 `manifest.json`
- [ ] zip 文件大小 < 100MB（通常应该 < 5MB）

## 🚀 提交准备

- [ ] 选择了合适的类别（生产力工具）
- [ ] 选择了可见性（公开/不公开/私有）
- [ ] 选择了分发地区
- [ ] 准备了单一用途说明
- [ ] 准备了权限说明
- [ ] 准备了数据使用说明

## ✅ 最终检查

- [ ] 再次测试所有功能
- [ ] 检查所有链接可访问
- [ ] 检查所有图片正确显示
- [ ] 阅读 Chrome Web Store 政策
- [ ] 确认没有违反政策的内容
- [ ] 准备好回答审核问题

---

## 🎯 提交流程

完成以上所有检查后：

1. 访问 [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
2. 点击"新增项"
3. 上传 `extension.zip`
4. 填写所有必需信息
5. 提交审核
6. 等待审核结果（1-3 个工作日）
