## 中文版本

### 概述

"还活着吗"（Are You Still Alive）是一个 Chrome 浏览器扩展程序，旨在通过有趣的方式帮助用户管理生命值，并在长时间未活跃时通知紧急联系人。我们非常重视您的隐私，本政策说明了我们如何收集、使用和保护您的个人信息。

### 1. 我们收集的信息

#### 1.1 用户活动数据

- **敲击记录**：您敲击木鱼的时间和次数
- **功德值**：累计的功德值数据
- **生命值（HP）**：当前和历史的 HP 数据
- **连续活跃天数**：您连续使用扩展的天数统计
- **每日统计**：每日敲击次数和活跃时间

#### 1.2 联系人信息

- **姓名**：紧急联系人的姓名
- **邮箱地址**：用于发送通知的邮箱
- **关系**：与您的关系（如家人、朋友）
- **优先级**：联系人的通知优先级
- **自定义消息**：您为联系人设置的个性化消息

#### 1.3 账号信息

- **Google 账号信息**：
  - 邮箱地址
  - 显示名称
  - 用户 ID（用于身份验证）
- **认证令牌**：用于维持登录状态的安全令牌

#### 1.4 设置和偏好

- **语言偏好**：界面语言设置（中文/英文）
- **通知设置**：通知阈值和偏好
- **邮件模板**：自定义的邮件模板内容

### 2. 信息的使用方式

我们使用收集的信息用于以下目的：

#### 2.1 提供核心功能

- 记录和显示您的敲击活动
- 计算和更新生命值（HP）
- 统计功德值和连续活跃天数
- 生成活动统计图表

#### 2.2 紧急通知服务

- 检测长时间未活跃状态
- 向紧急联系人发送通知邮件
- 提供邮件预览功能

#### 2.3 数据同步

- 在您的多个设备间同步数据
- 备份您的数据到云端
- 恢复数据到新设备

#### 2.4 改进服务

- 分析使用模式以改进功能
- 修复 bug 和技术问题
- 优化用户体验

### 3. 数据存储

#### 3.1 本地存储

- 使用 Chrome Storage API 在您的浏览器中本地存储数据
- 数据仅存储在您的设备上，除非您选择启用云端同步

#### 3.2 云端存储

当您使用 Google 账号登录时：

- 数据存储在 Google Firebase Firestore 数据库中
- 使用 Google Cloud Platform 的安全基础设施
- 数据中心位于 Google 指定的区域
- 所有数据传输使用 HTTPS 加密

#### 3.3 数据安全

- 传输加密：所有网络传输使用 TLS/SSL 加密
- 访问控制：使用 Firebase Security Rules 限制数据访问
- 身份验证：仅认证用户可以访问自己的数据
- 定期备份：云端数据定期自动备份

### 4. 数据共享

我们承诺：

#### 4.1 不会主动分享

- 我们**不会**将您的个人数据出售给第三方
- 我们**不会**将您的数据用于广告目的
- 我们**不会**与其他公司共享您的数据用于营销

#### 4.2 有限的数据共享

仅在以下情况下可能共享数据：

**紧急联系人通知**：

- 当您长时间未活跃时，我们会向您指定的紧急联系人发送邮件
- 邮件内容由您自定义，包含您希望分享的信息

**服务提供商**：

- Google Firebase：用于身份验证和数据存储
- Google Cloud Platform：用于云端基础设施
- 这些服务提供商受其自身隐私政策约束

**法律要求**：

- 遵守法律、法规或法律程序
- 保护我们的权利、隐私、安全或财产
- 防止欺诈或安全问题

### 5. 您的权利

## English Version

### Overview

"Are You Still Alive" is a Chrome browser extension designed to help users manage their life points in a fun way and notify emergency contacts during prolonged inactivity. We take your privacy seriously, and this policy explains how we collect, use, and protect your personal information.

### 1. Information We Collect

#### 1.1 User Activity Data

- **Knock Records**: Time and count of your wooden fish knocks
- **Merit Points**: Accumulated merit point data
- **Health Points (HP)**: Current and historical HP data
- **Consecutive Active Days**: Statistics of your consecutive usage days
- **Daily Statistics**: Daily knock counts and active time

#### 1.2 Contact Information

- **Name**: Names of emergency contacts
- **Email Address**: Email addresses for sending notifications
- **Relationship**: Your relationship with contacts (e.g., family, friends)
- **Priority**: Notification priority of contacts
- **Custom Messages**: Personalized messages you set for contacts

#### 1.3 Account Information

- **Google Account Information**:
  - Email address
  - Display name
  - User ID (for authentication)
- **Authentication Tokens**: Security tokens to maintain login status

#### 1.4 Settings and Preferences

- **Language Preference**: Interface language setting (Chinese/English)
- **Notification Settings**: Notification thresholds and preferences
- **Email Templates**: Customized email template content

### 2. How We Use Information

We use the collected information for the following purposes:

#### 2.1 Provide Core Features

- Record and display your knocking activity
- Calculate and update Health Points (HP)
- Track merit points and consecutive active days
- Generate activity statistics charts

#### 2.2 Emergency Notification Service

- Detect prolonged inactivity
- Send notification emails to emergency contacts
- Provide email preview functionality

#### 2.3 Data Synchronization

- Sync data across your multiple devices
- Backup your data to the cloud
- Restore data to new devices

#### 2.4 Service Improvement

- Analyze usage patterns to improve features
- Fix bugs and technical issues
- Optimize user experience

### 3. Data Storage

#### 3.1 Local Storage

- Uses Chrome Storage API to store data locally in your browser
- Data is only stored on your device unless you enable cloud sync

#### 3.2 Cloud Storage

When you log in with your Google account:

- Data is stored in Google Firebase Firestore database
- Uses Google Cloud Platform's secure infrastructure
- Data centers located in Google-designated regions
- All data transmission uses HTTPS encryption

#### 3.3 Data Security

- Transmission Encryption: All network transmissions use TLS/SSL encryption
- Access Control: Uses Firebase Security Rules to restrict data access
- Authentication: Only authenticated users can access their own data
- Regular Backups: Cloud data is automatically backed up regularly

### 4. Data Sharing

We promise:

#### 4.1 No Active Sharing

- We will **NOT** sell your personal data to third parties
- We will **NOT** use your data for advertising purposes
- We will **NOT** share your data with other companies for marketing

#### 4.2 Limited Data Sharing

Data may only be shared in the following situations:

**Emergency Contact Notifications**:

- When you are inactive for an extended period, we send emails to your designated emergency contacts
- Email content is customized by you and contains information you wish to share

**Service Providers**:

- Google Firebase: For authentication and data storage
- Google Cloud Platform: For cloud infrastructure
- These service providers are bound by their own privacy policies

**Legal Requirements**:

- Comply with laws, regulations, or legal processes
- Protect our rights, privacy, safety, or property
- Prevent fraud or security issues

### 5. Your Rights

#### 5.1 Right to Access

- You can view all your data in the extension at any time
- View account information in the "Settings" page
- View activity data in the "Statistics" page
- View contact information in the "Contacts" page

#### 5.2 Right to Modify

- You can modify your personal information at any time
- Edit contact information
- Update email templates
- Modify settings and preferences

#### 5.3 Right to Delete

- **Delete Local Data**: Click "Clear All Data" in the settings page
- **Delete Cloud Data**: Logging out will delete all cloud-stored data
- **Delete Account**: Contact us to request complete deletion of account and all related data

#### 5.4 Right to Export

- You can export your data (feature in development)
- Data provided in JSON format

#### 5.5 Withdraw Consent

- You can log out at any time to stop cloud synchronization
- Uninstalling the extension will delete all local data

### 6. Third-Party Services

This extension uses the following third-party services:

#### 6.1 Google Firebase

- **Purpose**: Authentication, data storage
- **Privacy Policy**: https://firebase.google.com/support/privacy
- **Data Collected**: Account information, user data

#### 6.2 Google Cloud Platform

- **Purpose**: Cloud infrastructure
- **Privacy Policy**: https://cloud.google.com/privacy
- **Data Collected**: Technical logs, usage data

These services have their own privacy policies, and we recommend you read and understand them.

### 7. Children's Privacy

This extension is not intended for children under 13 years of age. We do not knowingly collect personal information from children under 13. If you discover that a child has provided us with personal information, please contact us, and we will delete it immediately.

### 8. Data Retention

#### 8.1 Active Accounts

- We retain your data as long as you continue to use the extension
- Cloud data is retained continuously while you are logged in

#### 8.2 Inactive Accounts

- If you do not log in for more than 365 days, we may delete your cloud data
- We will attempt to notify you via email before deletion

#### 8.3 Account Deletion

- After account deletion, cloud data will be completely removed within 30 days
